# Instructions for AI Agents

## Overview

Java client library for [RabbitMQ](https://www.rabbitmq.com/) for the AMQP 1.0 protocol.

Consult the [API guide](https://www.rabbitmq.com/client-libraries/amqp-client-libraries) to learn about how this library is supposed to be used.

## Supported Java Versions

This client supports Java 11+.

## Build

```bash
./mvnw clean test-compile
```

## Sanity Check

```bash
./mvnw test-compile
```

### Running Tests

Tests require a RabbitMQ node on `localhost:5672` with the default `localhost`-scoped credentials (`guest`/`guest`):

```bash
ci/start-broker.sh
```

```bash
./mvnw verify -Drabbitmqctl.bin=DOCKER:rabbitmq
```

To run a single test:

```bash
./mvnw test -Drabbitmqctl.bin=DOCKER:rabbitmq -Dtest=ManagementTest
```

## Repository Layout

* `src/main/java/com/rabbitmq/client/amqp`: the public API
* `src/main/java/com/rabbitmq/client/amqp/impl`: internal packages
* `src/main/qpid/org/apache/qpid/protonj2`: fork of Apache QPid ProtonJ2 for the AMQP 1.0 layer
* `src/test/java`: tests

## Key Files

Under `src/main/java/com/rabbitmq/client/amqp/`:

* `Environment.java`: entry point for creating connections
* `Connection.java`, `Publisher.java`, `Consumer.java`: key interfaces
* `impl/AmqpEnvironmentBuilder.java`, `impl/AmqpEnvironment.java`: implementation classes to bootstrap the system
* `impl/AmqpConnection.java`, `impl/AmqpPublisher.java`, `impl/AmqpConsumer.java`: key implementation classes

## After Completing a Task

### Iterative Reviews

After completing a task, perform up to twenty iterative reviews of your changes.
In every iteration, look for meaningful improvements that were missed, for gaps in test coverage, and for deviations from the instructions in this file.

If no meaningful improvements are found for three iterations in a row, report it and stop iterating.
